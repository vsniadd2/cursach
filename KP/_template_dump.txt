Особенностью проекта является сочетание карты парковок, бронирования в реальном времени, одноразового QR-кода и системы баллов.

P119|Стиль2|2.3. Описание программных средств
P120|Курсовое_Стандарт|Проект основан на современном технологическом стеке, включающем ASP.NET Core Web API на стороне backend, Next.js и React для frontend-разработки, TypeScript для типизации клиентской части, а также Tailwind CSS для оформления интерфейса. Для realtime-обновлений подготовлен SignalR-хаб. Для отображения карты используется подход с подключением картографической библиотеки, такой как Leaflet или Mapbox. Для отображения QR-кода на клиенте может применяться React-компонент генерации QR по данным, полученным с backend.
P121|Курсовое_Стандарт|Такой выбор технологий обеспечивает масштабируемость, кроссплатформенность и удобство разработки. ASP.NET Core позволяет реализовать защищенные API, JWT-авторизацию и интеграцию с SignalR. Next.js обеспечивает современную маршрутизацию, серверные обработчики запросов, быстрый frontend и возможность дальнейшей SEO-оптимизации главной страницы.
P122|Курсовое_Стандарт|В качестве инструментов разработки используются IDE Visual Studio Code и JetBrains Rider, система контроля версий Git, пакетный менеджер npm и .NET SDK. Проект организован как монорепозиторий, где frontend находится в папке apps/web, backend – в services/api/Xpark.Api, а общие типы и контракты могут храниться в packages/shared.
P123|Курсовое_Стандарт|Среди возможных сложностей можно выделить обеспечение корректного бронирования одного места несколькими пользователями, защиту платежных данных, надежную работу QR-кодов, синхронизацию данных в реальном времени и переход от демонстрационного in-memory хранилища к полноценной базе данных.

P279|Курсовое_Стандарт|Для начала работы с платформой необходимо открыть главную страницу приложения и нажать кнопку регистрации. Если аккаунт уже создан, необходимо перейти на страницу авторизации, указать email и пароль (см. рис. 2.2), после чего нажать кнопку входа.

P281|Курсовое_Стандарт|После входа в систему пользователь может перейти в раздел «Найти парковку» (см. рис. 2.6). На странице отображается список парковок и карта с метками.
