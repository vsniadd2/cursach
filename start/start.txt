ЗАПУСК ПРОЕКТА (кратко)



1) Docker (PostgreSQL + API, ПК)

   docker compose up -d --build

   Из корня: expogo-cursach

   API: http://localhost:5278



2) Backend .NET (ПК, если запускаете без Docker API)

   cd backend

   dotnet run

   API: http://localhost:5278



3) Frontend Expo (ПК → выбор устройства)

   cd frontend

   npm start



   Перед стартом выполняется scripts/seed.js (prestart): логин admin и вызов /seed/run.

   Учётные записи по умолчанию (общая организация Экспого):

     • admin / 123456 — администратор (управление пользователями, аудит, интеграции…)

     • user  / 123456 — обычный пользователь (клиенты, сделки, задачи)

   Регистрация всегда создаёт обычного пользователя (Member) в той же организации.

   Адрес API: http://localhost:5278 (захардкожен в seed.js и src/api/config.ts).



   • Телефон (iOS/Android): приложение Expo Go, тот же Wi‑Fi что ПК → Scan QR из Expo Go.

   • Браузер на этом же ПК: npm run web (или в меню Expo клавиша w)

   • Эмулятор Android: npm run android  (или клавиша a)

   • Симулятор iOS: только на macOS — npm run ios  (или клавиша i)



Порядок: обычно 1 → 3. Пункт 2 нужен только если API запускается локально без Docker.



База данных: API при старте выполняет MigrateAsync() и seed admin/user. Если сменили схему или старые данные мешают:

   docker compose down -v

   docker compose up -d --build

   cd frontend && node scripts/seed.js



Настройки: БД, JWT, DefaultTenant и SeedUsers — в appsettings*.json. Файлов .env нет.



Деньги: суммы сделок в БД в USD; в интерфейсе — в валюте профиля (по умолчанию BYN).
