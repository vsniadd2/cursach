Описание структуры клиентской части проекта (frontend)

В папке src/theme:
1)	colors.ts – Цветовая палитра приложения (единые цвета для всех экранов).

В папке src/utils:
2)	calendarRu.ts – Утилиты для календаря/дат (форматирование на русском, расчёт недели и т.п.).
3)	appAlerts.ts – Вспомогательные функции для отображения уведомлений/Alert (если используются на экранах).

В папке src/api:
4)	config.ts – Настройка базового адреса API (для Expo Go может определяться по IP хоста, чтобы работало на телефоне).
5)	client.ts – Формирование полного URL запроса к API на основе базового адреса.
6)	requests.ts – Унифицированные функции запросов к серверу (получение/создание/обновление/удаление) + обработка ошибок.
7)	types.ts – Типы данных (DTO) для ответов сервера: дашборд, клиенты, задачи, сделки и т.д.

В папке src/auth:
8)	AuthContext.tsx – Контекст авторизации (интерфейс состояния и методов авторизации).
9)	AuthProvider.tsx – Провайдер авторизации: логин/регистрация, обновление токена, запросы к API с токеном.
10)	AuthShell.tsx – UI-оболочка экранов входа/регистрации (фон, карточка, кнопки, адаптация под iOS/web).
11)	tokens.ts – Хранение токенов в AsyncStorage (загрузка/сохранение/очистка).

В папке src/components:
12)	AppHeader.tsx – Шапка приложения (логотип/название), используется на основных экранах после входа.
13)	MainTabBar.tsx – Нижняя панель вкладок (Главная, Клиенты, Сделки, Задачи, Ещё) и навигация между разделами.

В папке src/navigation:
14)	RootNavigator.tsx – Корневая навигация: разделение на поток авторизации и основной интерфейс приложения.
15)	types.ts – Типы навигации (параметры экранов) для React Navigation.
16)	DealsStackNavigator.tsx – Навигационный стек раздела «Сделки» (воронка, карточка клиента, редактирование сделки).
17)	ClientsStackNavigator.tsx – Навигационный стек раздела «Клиенты» (список, карточка клиента, редактирование клиента).

В папке src/screens:
18)	LoginScreen.tsx – Экран авторизации пользователя (ввод логина/пароля, вход в систему).
19)	RegisterScreen.tsx – Экран регистрации (создание нового аккаунта).
20)	DashboardScreen.tsx – Главная страница (дашборд): метрики и лента активности.
21)	ClientsScreen.tsx – Раздел «Клиенты»: список клиентов + поиск + переход в карточку.
22)	ClientDetailScreen.tsx – Карточка клиента: подробная информация и связанные данные.
23)	ClientEditScreen.tsx – Создание/редактирование клиента (форма + сохранение/удаление).
24)	DealsPipelineScreen.tsx – Раздел «Сделки»: воронка по стадиям, просмотр и смена стадии.
25)	DealEditScreen.tsx – Создание/редактирование сделки (форма параметров сделки).
26)	TasksScreen.tsx – Раздел «Задачи»: задачи на выбранную дату, статистика, отметка выполнения.
27)	TaskEditScreen.tsx – Создание/редактирование задачи (форма задачи).
28)	MoreScreen.tsx – Раздел «Ещё»: дополнительные пункты/информация.

Описание структуры серверной части проекта (backend)

В корне папки backend:
1)	Program.cs – Точка входа сервера и конфигурация приложения (контроллеры, CORS, JWT, JSON).
2)	appsettings.json – Основные настройки (строка подключения к PostgreSQL, параметры JWT, логирование).
3)	appsettings.Development.json – Настройки для режима разработки (например, сид-пользователь).
4)	ExpogoCrm.Api.csproj – Файл проекта .NET (зависимости и таргет-фреймворк).

В папке backend/Data:
5)	Entities.cs – Описание сущностей предметной области (пользователи, клиенты, сделки, задачи, события) и перечислений (стадии сделок, приоритеты задач).
6)	ExpogoDbContext.cs – Контекст EF Core: DbSet’ы таблиц, связи между сущностями, индексы и ограничения целостности.

В папке backend/Auth:
7)	AuthModels.cs – DTO-модели для запросов/ответов авторизации (логин, регистрация, refresh и т.п.).
8)	JwtTokenService.cs – Сервис генерации JWT access-token и вспомогательная логика для refresh-token.
9)	PasswordHasher.cs – Хэширование и проверка паролей (пароли в БД не хранятся в открытом виде).

В папке backend/Controllers:
10)	AuthController.cs – Авторизация и регистрация пользователей, выдача и обновление токенов.
11)	ClientsController.cs – Работа с клиентами: список/поиск, карточка клиента, создание/редактирование/удаление.
12)	DealsController.cs – Работа со сделками: список, создание/редактирование/удаление, смена стадии, данные для воронки.
13)	TasksController.cs – Работа с задачами: список на дату, создание/редактирование/удаление, отметка выполнения.
14)	DashboardController.cs – Дашборд: сводные показатели и базовая аналитика по сделкам и задачам.
15)	SeedController.cs – Демонстрационные данные для быстрого тестирования (заполняет БД при необходимости).
16)	WeatherForecastController.cs – Тестовый контроллер (пример/шаблон, не критичен для CRM-логики).
