ТЕСТИРОВАНИЕ И АНАЛИЗ ПОЛУЧЕННЫХ РЕЗУЛЬТАТОВ

Тестирование программного обеспечения (Software Testing) — проверка соответствия между реальным и ожидаемым поведением программы, осуществляемая на конечном наборе тестов, выбранном определённым образом.

Объект тестирования — клиентская часть CRM (Expo / React Native) и серверная часть REST API на ASP.NET Core с базой PostgreSQL, описанные в структуре проекта (экраны авторизации, главная, клиенты, сделки, задачи, раздел "Ещё").

Над программой были проведены следующие тесты:
1) Тест №1. Проверка требований валидации полей в формах авторизации/регистрации.
2) Тест №2. Проверка успешной авторизации.
3) Тест №3. Проверка на корректное отображение элементов интерфейса.
4) Тест №4. Проверка создания записи в разделе "Клиенты".
5) Тест №5. Проверка загрузки и отображения аналитики на главной странице (дашборд).
6) Тест №6. Проверка раздела "Сделки": воронка по стадиям и работа с карточками.
7) Тест №7. Проверка информирующих и подтверждающих диалогов при действиях пользователя.

Тест №1. На экране регистрации пользователь вводит в поле "Email" некорректное значение (без "@") и отправляет форму. Ожидаемый результат: отображается сообщение "Некорректный email". Фактический результат совпал с ожидаемым.

Примечание: на экране входа используется поле "Логин", а не email; проверка формата email относится к сценарию регистрации (и к опциональному полю email при создании пользователя).

Анализ полученных результатов

По результатам тестирования подтверждена корректная работа: регистрации, авторизации, навигации, разделов "Клиенты" и "Сделки", дашборда, а также диалогов подтверждения. Приложение пригодно к демонстрации в составе "клиент Expo Go + backend .NET + PostgreSQL".
